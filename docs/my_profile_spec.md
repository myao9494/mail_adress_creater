# 仕様書 - マイプロフィールかんたん入力コピー機能

## 1. 概要
社内システムや各種フォーム等への入力作業を効率化するため、ユーザー個人の「メールアドレス、電話番号、住所、部署、家アド」を事前にアプリへ登録しておき、メイン画面上の小さなボタンからワンクリックでクリップボードへコピーできるようにする機能です。
オートフィル機能が制限されている会社のPC環境などにおいて、素早く正確なデータ入力を支援することを目的としています。

## 2. 画面設計 (UI)

### 2.1. 設定画面（ハンバーガーメニュー内）
ハンバーガーメニュー（設定メニュー）を開くと、従来のシステム設定（キーワード、同期間隔など）の下に「プロフィール設定」の行が追加されます。
- **メールアドレス**: テキスト入力欄（`your.email@example.com` 等）
- **電話番号**: テキスト入力欄（`090-0000-0000` 等）
- **住所**: テキスト入力欄（`住所を入力してください` 等）
- **部署**: テキスト入力欄（`部署を入力してください` 等）
- **家アド**: テキスト入力欄（`home.email@example.com` 等）

「設定保存」ボタンを押すことで、これらの値がバックエンドに送信され SQLite の `settings` テーブルへ永続化されます。

### 2.2. コピーボタン（メイン画面ヘッダー部）
ヘッダーの「お気に入り」ボタンの右側に、区切り線（縦線）を挟んで 5つのプロフィールコピーボタンが追加されます。
- **[メール]** / **[電話]** / **[住所]** / **[部署]** / **[家アド]**
- サイズ: `text-[10px]` の極小サイズ（使用頻度が小さいため、控えめな配置）
- スタイル: ダークテーマに合わせたダークグレー背景（`bg-gray-800 hover:bg-gray-700`）

## 3. 挙動・コピーロジック

1. コピーボタンをクリックします。
2. 設定された値が存在する場合：
   - 設定値をトリム（前後の余白削除）した上で、クリップボードにコピーします。
   - 画面下部に「メールアドレスをコピーしました」や「家のメールアドレスをコピーしました」等の通知トーストを一時表示します。
3. 設定された値が空（または未設定）の場合：
   - クリップボードへのコピーは行わず、「メールアドレスが未設定です。設定メニューから登録してください」という警告メッセージをトーストで表示します。

## 4. データベース構成 (SQLite)
`settings` テーブルに以下のキーと値のペアを追加して管理します。
- `my_email`: メールアドレス文字列
- `my_phone`: 電話番号文字列
- `my_address`: 住所文字列
- `my_dept`: 部署文字列
- `my_home_email`: 家のメールアドレス文字列

値はすべて設定保存時に自動的に前後の余白がトリムされます。
デフォルト初期値はすべて空文字列 `""` です。
