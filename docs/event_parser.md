# 予定解析機能仕様書 (event_parser)

自然言語（日本語）で入力された予定テキストから、Outlook予定の作成に必要な情報を自動抽出する機能の仕様書です。

## 主要機能
入力された自然文テキストから以下の項目を抽出・生成します：
1. **開始日時 (Start)**
2. **終了日時 (End)**
3. **所要時間 (Duration)**
4. **予定件名 (Subject)**
5. **場所 (Location)**
6. **終日予定フラグ (All Day)**

---

## 処理フロー

予定解析は `backend/event_parser.py` の `parse_event_text` を起点に、以下の手順で実行されます。

```
[入力テキスト]
      │
      ▼
1. テキストの正規化 (normalize_text)
   - 全角英数記号の半角化
   - 連続する空白文字を1つの半角スペースに縮退
      │
      ▼
2. 日付の抽出 (extract_date)
   - 相対日付（明日、明後日、今日、X日後、X週間後）の判定
   - 曜日指定（今週の木曜日、来週の月曜日など）の判定
   - 具体的な年月日（2026年 5 月 28 日 (木) など）の判定
     ※ 数字と単位（年/月/日）の間のスペースや、日付直後の曜日表記も許容
   - 抽出した日付部分を元のテキストから除去
      │
      ▼
3. 時間範囲の抽出 (extract_time_range)
   - コロン（:）前後の余分なスペースを排除 (例: "15 : 00" -> "15:00")
   - 時間範囲を表すキーワード（〜、-、~、から、より）と前後時間の抽出
   - 抽出した時間範囲を元のテキストから除去
      │
      ▼
4. 件名と場所の抽出 (extract_subject_and_location)
   - 接頭辞（から、に、は、の、が、を、へ）のトリミング
   - 「場所」「会場」「於」「@」に続く文字列を場所として抽出
   - 残りのテキストを件名（Subject）として決定
      │
      ▼
5. 予定オブジェクトの生成 (ParsedEvent)
   - Outlookに送信可能な形式でデータを返却
```

## 日付・時間解析の許容フォーマット（抜粋）

### 日付表記
- **相対値**: `今日`, `明日`, `あした`, `明後日`, `あさって`, `3日後` (数字との間のスペースも許容), `2週間後`
- **曜日**: `木曜日`, `今週の水曜`, `来週の金曜日` (「の」を省略した `来週木曜日` も許容)
- **絶対日付**:
  - `2026/05/28`
  - `2026.5.28` (ドット区切りを許容)
  - `5-28`
  - `2026年 5 月 28 日` (数字と単位の間のスペースを許容)
  - `2026年 5 月 28 日 (木)` (日付直後の括弧付き曜日表記を許容・自動除去)
  - `5/28木` (括弧なしで日付の直後に曜日が連結した表記も許容・自動除去)

### 時間表記
- `15:00-16:30` (各種ハイフン `ー`, `−`, `–`, `—` を許容)
- `15 : 00 〜 16 : 00` (コロン前後のスペースや各種波線 `～`, `~` を許容)
- `１５　：　３０` (全角スペースを含む全角コロン表記を許容)
- `14:00` (開始時間のみの場合は自動で1時間の予定として設定)
- `15時半` / `15 時 半` (数字と単位の間のスペースを許容し `15:30` として解釈)
- `15時` / `15時00分` / `15 時 30 分` (スペース入りの時分表記も許容)
- `午後3時` / `午後 3:00` / `夕方 5 時` / `夜 8:30` (多様な時間帯修飾表記を許容)
- `正午` -> `12:00` として解釈

### 場所表記
- `会議 @ 会議室A` / `会議 @会議室A` (`@` 記号に続く場所の抽出、前後のスペースを許容)
- `於 会議室B 会議` / `於会議室B 会議` (漢字「於」に続く場所の抽出)
- `場所: 会議室C 会議` / `会場： オンライン 会議` (「場所:」「会場：」に続く場所の抽出)
- **境界判定**: 場所の後にスペース区切りで件名が続く場合（例: `於 会議室B 会議`）、スペースの位置を境界として場所名（`会議室B`）と件名（`会議`）を正確に分離抽出します。

## 開発・変更履歴
- **2026-05-22**:
  - コロン前後にスペースが含まれる時間表記（例: `15 : 00`）に対応。
  - 日付直後に曜日が記述されている表記（例: `2026年 5 月 28 日 (木)`）で曜日部分も適切に日付と一緒に除去されるよう正規表現を改善。
  - **テストケースの網羅的拡充**: 人間が入力する多様なゆらぎ（ドット区切りの日付、スペース入りコロン・時分、午後/夕方/夜/正午修飾、スペース区切りの場所表記など）に対する網羅的なテストケースを追加し、パースロジックの堅牢性を大幅に強化。
