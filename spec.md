Outlook宛先作成アプリ 仕様書

1. 目的

Outlookの送信履歴から抽出した「送信先名 × 送信回数」のCSVデータを利用し、
	•	宛先（To）
	•	CC

を効率よく作成し、Outlookに貼り付けるための軽量なフロントエンド専用アプリを作成する。

人間が最終確認・取捨選択を行う前提とし、自動化しすぎないことを重視する。

⸻

2. 入力データ仕様

2.1 CSVファイル
	•	ファイル名：send_mail-ranking_tabulator.csv
	•	配置場所：フロントエンドの public/ 直下
	•	読み込み方法：fetch() による静的読み込み
	•	再読み込み方法：ブラウザのリロード

2.2 CSVフォーマット

列名	型	説明
名前	string	Outlookに貼り付ける表示名（メールアドレスは含まない）
回数	number	送信回数（表示・ソート用）

2.3 テストデータ例

名前,回数
山田 太郎,120
山田 花子,85
佐藤 一郎,200
佐藤 次郎,60
鈴木 恒一,150
鈴木 恒一（開発）,40
高橋 真紀,95
伊藤 健,30

3. 画面構成

3.1 全体レイアウト
	•	縦長画面での利用を想定
	•	横幅はノートPC画面の半分程度
	•	横スクロールは発生させない
	•	2ペイン横並び構成

+-----------------------------+
|  宛先（To） |   CC          |
|-------------|---------------|
| 検索BOX     | 検索BOX       |
|-------------|---------------|
| [✓] 山田…  | [✓] 佐藤…    |
| [✓] 鈴木…  | [✓] 高橋…    |
|             |               |
| 宛先作成   | CC作成        |
+-----------------------------+


⸻

4. ペイン共通仕様（宛先 / CC）

4.1 検索機能
	•	入力方式：テキスト入力
	•	区切り文字：半角スペース
	•	検索方式：OR検索
	•	検索条件：部分一致
	•	大文字・小文字：区別しない
	•	Enterキーで検索実行

検索例

山田 鈴木

→ 「山田」または「鈴木」を含む行を抽出

⸻

4.2 検索結果表示
	•	表示項目
	•	チェックボックス
	•	名前
	•	回数（右寄せ・小さめ表示）
	•	並び順
	•	回数の降順
	•	初期状態
	•	検索結果はすべてチェックON
	•	想定操作
	•	同姓・同名などを人間が目視確認
	•	不要な項目のチェックをOFF

⸻

4.3 宛先作成ボタン
	•	ボタン名称
	•	左ペイン：宛先作成
	•	右ペイン：CC作成
	•	動作仕様
	1.	チェックONの「名前」を取得
	2.	;（セミコロン）で連結
	3.	クリップボードにコピー
	4.	コピー成功時に軽いフィードバック表示（例：トースト）

出力例

山田 太郎;鈴木 恒一;佐藤 一郎


⸻

5. 技術仕様

5.1 フロントエンド
	•	React（関数コンポーネント）
	•	バックエンドなし
	•	状態管理はReact標準（useState / useEffect）を使用

5.2 状態管理方針（概念）
	•	CSV全件データ
	•	宛先（To）用検索結果とチェック状態
	•	CC用検索結果とチェック状態

※ 宛先とCCは完全に独立した状態を持つ

⸻

6. ダークモード
	•	デフォルトでダークモードを使用
	•	OS連動は必須としない
	•	配色方針
	•	背景：濃いグレー〜黒
	•	文字色：白〜薄いグレー
	•	ボタン・チェックボックス：視認性重視

⸻

7. 対応環境
	•	開発環境：macOS
	•	利用環境：Windows
	•	ブラウザ：Chrome / Edge
	•	Outlook：デスクトップ版

⸻

8. 非対応事項（スコープ外）
	•	メールアドレスの直接管理
	•	Outlook API 連携
	•	CSVアップロードUI
	•	データの永続化
	•	ユーザー認証・権限管理

⸻

9. 補足

本アプリは「人間の判断を前提とした補助ツール」であり、
完全自動化ではなく、確認しやすさ・軽快さを最優先とする。
